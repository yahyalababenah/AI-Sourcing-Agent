# تقرير: دورة حياة العميل والمندوب
**AI-Sourcing Hub — تاريخ: 2026-06-29**

---

## المشكلة التي يحلّها المشروع

المستورد العربي يريد شراء منتجات من الصين لكنه لا يعرف التكلفة الحقيقية حتى ينتهي من كل مفاوضاته — وذلك يسبب:
- أخطاء تقدير تصل لـ 22% في التكلفة النهائية
- إلغاء 15% من الصفقات بسبب مفاجآت الجمارك والشحن
- دورة استيراد تستغرق 14–21 يوماً بدلاً من أيام

**القيمة الأساسية للمنصة:** العميل يرى التكلفة الكاملة (FOB + جمارك + ضريبة) قبل أن يتواصل مع أي أحد.

## دورة حياة المندوب (Agent)

المندوب هو المحور — يربط بين الموردين الصينيين والعملاء المستوردين.

```
المرحلة ١: الإعداد
─────────────────
أ. Admin يضيف المندوب في النظام (لا تسجيل مفتوح في المرحلة الأولى)
ب. المندوب يسجّل دخوله ويكمّل ملفه الشخصي

المرحلة ٢: بناء الكاتالوج
──────────────────────────
أ. يضيف موردين صينيين (الاسم، الموقع، التخصص)
ب. يرفع كاتالوج المورد (PDF أو صورة)
ج. النظام يستخرج المنتجات تلقائياً (OCR + LLM)
د. المنتجات تُزامَن تلقائياً في صفحة البحث العامة
هـ. المندوب يراجع ويصحّح المنتجات إذا احتاج

المرحلة ٣: معالجة الطلبات
────────────────────────────
أ. يستلم إشعار: عميل طلب عرض سعر على منتج
ب. يفتح الطلب ويضيف تكلفة الشحن الفعلية
   (الجمارك والضريبة محسوبة تلقائياً — هو يضيف الشحن فقط)
ج. النظام يولّد PDF بعرض السعر الكامل
د. يرسل العرض للعميل

المرحلة ٤: متابعة الطلب
──────────────────────────
أ. العميل يوافق → المندوب يؤكد الطلب مع المورد
ب. يحدّث حالة الشحنة بشكل دوري
ج. العميل يتابع الشحنة من حسابه
```

## دورة حياة العميل (Buyer)

```
المرحلة ١: الاكتشاف
────────────────────
أ. يسجّل في المنصة
ب. يبحث عن منتج بالاسم أو الوصف (عربي أو إنجليزي)
ج. يرى نتائج من كاتالوجات الموردين المستخرجة

المرحلة ٢: التقييم والتكلفة ← القيمة الأساسية
────────────────────────────────────────────────
أ. يضغط على منتج يعجبه
ب. يرى التكلفة التقديرية الكاملة:
   - السعر الأساسي (FOB) بالدينار الأردني
   - الجمارك والرسوم (محسوبة تلقائياً)
   - ضريبة القيمة المضافة
   - المجموع التقديري
   - "الشحن يُحدَّد بعد تأكيد الكمية مع المندوب"
ج. يُدخل الكمية ويرى الحساب يتحدث فورياً

المرحلة ٣: طلب عرض السعر
───────────────────────────
أ. يضغط "اطلب عرض سعر رسمي"
ب. يدخل: الكمية، ميناء الوصول، ملاحظات
ج. الطلب يصل للمندوب تلقائياً

المرحلة ٤: الاستلام والموافقة
────────────────────────────────
أ. ينتظر رد المندوب (يرى حالة الطلب)
ب. يستلم PDF بعرض السعر النهائي (مع الشحن الفعلي)
ج. يوافق أو يرفض أو يتفاوض عبر الشات

المرحلة ٥: متابعة الشحنة
──────────────────────────
أ. بعد الموافقة: يتابع حالة الشحنة من حسابه
ب. يستلم إشعارات عند تحديث الحالة
```

### ❌ ما ينقص ويجب إنجازه

#### الأولوية الأولى — ربط الـ RFQ بالمنتج المحدد
**المشكلة:** عندما يطلب العميل عرض سعر، الـ RFQ لا يحمل رابطاً مباشراً لمعرّف المنتج في `catalog_products`. البيانات تُرسل كنص داخل `extracted_entities` فقط.

**المطلوب:**
- إضافة `catalog_product_id` (UUID) كحقل في جدول `rfqs` أو في جدول `products` الفرعي
- تعديل الـ Modal ليرسل هذا الـ ID مع الطلب
- يساعد المندوب لاحقاً على رؤية المنتج المطلوب بدقة

**الملفات:** `app/modules/intake/models.py`، `app/modules/intake/schemas.py`، `frontend/src/pages/catalog/MarketplacePage.tsx`

---

#### الأولوية الثانية — لوحة تحكم المندوب للـ RFQs الواردة
**المشكلة:** المندوب لا يملك صفحة واضحة تعرض له الطلبات الواردة ويتيح له إضافة الشحن وإرسال العرض.

**المطلوب:**
- صفحة "الطلبات الواردة" للمندوب: قائمة بالـ RFQs مع الحالة
- صفحة تفاصيل الطلب: تعرض المنتج + التكلفة المحسوبة + حقل لإدخال الشحن
- زر "أرسل العرض" يولّد PDF ويرسله للعميل تلقائياً

**الملفات:** `frontend/src/pages/rfq/`، `app/modules/intake/router.py`، `app/modules/output/`

---

#### الأولوية الثالثة — لوحات التحكم المخصصة لكل دور
**المشكلة:** لوحة التحكم الحالية عامة ولا تعكس أولويات كل مستخدم.

**المطلوب:**

*للعميل:*
- "طلباتي" — قائمة بكل RFQs أرسلها مع حالتها
- "عروض السعر المستلمة" — مع إمكانية المقارنة والموافقة
- "شحناتي" — تتبع مباشر

*للمندوب:*
- "الطلبات الواردة" — تنتظر ردي
- "الكاتالوجات المرفوعة" — مع حالة الاستخراج
- "الموردون" — قائمة الموردين المضافين

**الملفات:** `frontend/src/pages/dashboard/`

---

#### الأولوية الرابعة — إشعارات الوقت الفعلي
**المشكلة:** لا يوجد نظام إشعارات — المندوب لا يعرف أن عميلاً طلب عرض سعر إلا لو فتح الصفحة يدوياً.

**المطلوب:**
- إشعار للمندوب عند وصول RFQ جديد
- إشعار للعميل عند رد المندوب بعرض سعر
- يمكن تنفيذها عبر SSE (موجود في الـ chat module) أو Celery + بريد إلكتروني

**الملفات:** `app/modules/chat/router.py` (SSE موجود)، `app/modules/intake/`

---

#### الأولوية الخامسة — الصفحات الشخصية للمندوب والعميل
**المشكلة:** لا يوجد حالياً صفحة شخصية (Profile) واضحة لأي من الدورين — المستخدم لا يستطيع رؤية معلوماته أو تعديلها بسهولة.

**المطلوب:**

*صفحة المندوب الشخصية:*
- الاسم، الصورة الشخصية، رقم الهاتف، البريد الإلكتروني
- نبذة مختصرة عنه وعن خدماته
- عدد الموردين المضافين، عدد الكاتالوجات المرفوعة، عدد الطلبات المنجزة
- إمكانية تعديل المعلومات

*صفحة العميل الشخصية:*
- الاسم، رقم الهاتف، البريد الإلكتروني
- ميناء الوصول المفضّل (يُحفظ افتراضياً في كل طلب)
- العملة المفضّلة (JOD / USD)
- عدد الطلبات المرسلة، عدد الصفقات المكتملة
- إمكانية تعديل المعلومات

**الملفات:** `frontend/src/pages/settings/`، `app/modules/auth/router.py`، `app/modules/auth/schemas.py`

---

#### الأولوية السادسة — مراجعة المنتجات المستخرجة (Human-in-the-loop)
**المشكلة:** المندوب لا يملك واجهة سهلة لمراجعة وتصحيح المنتجات بعد الاستخراج قبل ظهورها للعملاء.

**المطلوب:**
- في صفحة تفاصيل المستند: جدول قابل للتعديل للمنتجات المستخرجة
- زر "نشر في الكاتالوج" يُفعّله المندوب بعد المراجعة (بدلاً من النشر التلقائي)

**الملفات:** `frontend/src/pages/documents/DocumentDetailPage.tsx`، `app/modules/catalog/`

## خريطة الطريق المقترحة

```
الأسبوع الأول   → الأولوية ١ + ٢ (ربط RFQ + لوحة المندوب)
الأسبوع الثاني  → الأولوية ٣ + ٥ (لوحات التحكم + الصفحات الشخصية)
الأسبوع الثالث  → الأولوية ٤ (الإشعارات)
الأسبوع الرابع  → الأولوية ٦ + اختبار الرحلة الكاملة
```

---

## اختبار الرحلة الكاملة (Definition of Done)

المشروع يُعتبر جاهزاً عندما يمكن إكمال هذه الرحلة بدون تدخل تقني:

1. **المندوب** يرفع كاتالوج صيني → يرى المنتجات مستخرجة → يراجعها → ينشرها
2. **العميل** يبحث عن "إضاءة صناعية" → يرى المنتجات مع التكلفة التقديرية بالدينار
3. **العميل** يختار منتج ويطلب 500 وحدة → الطلب يصل للمندوب
4. **المندوب** يضيف تكلفة الشحن → النظام يولّد PDF → يُرسل للعميل
5. **العميل** يوافق → يتابع حالة الشحنة

هذه الخطوات الخمس هي المنتج الحقيقي.
