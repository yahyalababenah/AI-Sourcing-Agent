> تاريخ التقرير: 2026-07-01 · الفرع الحالي: `fix/performance-optimization`

## 1. نظرة عامة

المنصة عبارة عن B2B Sourcing Hub عام (وليس مقتصراً على قطاع الإضاءة الصناعية كما كان مذكوراً سابقاً في بعض الوثائق القديمة) — تم التأكد من الكود أن أي إشارة لقطاع "الإضاءة الصناعية" كتحديد نطاق (scoping) **أُزيلت بالكامل**. محرك المطابقة (`app/modules/intake/matcher.py`) يدعم حالياً **10 فئات منتجات عامة**: العناية الشخصية، الأغذية والمشروبات، المنسوجات، الإلكترونيات، الآلات، المعادن والأدوات، الأثاث، التغليف، الكيماويات، الزجاج والسيراميك. الكلمة "إضاءة" لا تظهر إلا كمثال واحد ضمن قائمة أمثلة عامة في prompt الخاص بالـ OCR، وليست قيداً فعلياً.

**القيمة الأساسية:** يرى العميل التكلفة الكاملة المقدّرة (سعر أساسي + شحن + جمارك + ضريبة) لأي منتج من أي فئة قبل التواصل مع مندوب، ثم يطلب عرض سعر رسمي يمر عبر محرك مطابقة تلقائي مع الموردين.

## 3. المميزات التي تم إنجازها فعلياً وتعمل (مُتحقق منها في الكود)

### أ. الراوترات الخلفية المسجَّلة فعلياً في `app/main.py` (9 موديولات + endpoints مساعدة)

| الراوتر | المسار | ما يفعله فعلياً |
|---------|--------|------------------|
| `auth` | `/api/v1/auth` | تسجيل/دخول/تجديد توكن/الملف الشخصي/تسجيل خروج — JWT كامل |
| `intake` | `/api/v1/intake` | CRUD لطلبات RFQ، ترجمة النص، تشغيل المطابقة، صندوق مطابقات المورد، البوابة العامة للـ RFQs، استيراد دفعي |
| `documents` | `/api/v1/documents` | رفع/عرض/حذف المستندات، تشغيل معالجة OCR، عرض/تعديل العناصر المستخرجة |
| `pricing` | `/api/v1/pricing` | CRUD لقواعد التسعير + السجل التاريخي، حساب التكلفة الكاملة، تقدير سريع، تحديث سعر الصرف |
| `output` (quotes) | `/api/v1/quotes` | إنشاء/عرض عروض الأسعار، توليد PDF غير متزامن، تحديث الحالة، قبول/رفض العميل، تتبع الشحنة |
| `catalog` | `/api/v1/catalog` | بحث/عرض منتجات السوق العام، قائمة انتظار المراجعة، الموافقة/الرفض |
| `chat` | `/api/v1/chat` | إنشاء/عرض غرف الشات، إرسال/عرض الرسائل، بث SSE للرسائل الحية |
| `notifications` | `/api/v1/notifications` | بث SSE حقيقي (`/stream`) للإشعارات الفورية |
| `monitoring` (admin) | `/api/v1/admin` | إحصائيات تكلفة الـ AI، إحصائيات النظام، إدارة المستخدمين، حالة تحقق الموردين |

بالإضافة إلى: `/health` (فحص متوازٍ لقاعدة البيانات + Redis + MinIO + Celery + LLM)، `/metrics` (Prometheus).

### ب. صفحات الواجهة الأمامية الفعلية (بحسب محتوى الكومبوننتات وليس أسماء الملفات فقط)

- **admin/** — لوحة مراقبة حية (صحة النظام، رسوم بيانية لأزمنة الاستجابة، حالة circuit breaker)، وصفحة تحقق من الموردين مع إجراء موافقة فعلي
- **auth/** — دخول بتبويبات حسب الدور (عميل/مندوب/مورد)، تسجيل، ودخول أدمن منفصل
- **catalog/** — سوق عام قابل للبحث والفلترة مع تقدير سعر سريع، **صفحة مراجعة المنتجات (ProductReviewPage) فعّالة فعلاً** (موافقة/رفض)، صفحات منتجات المورد وواجهة عرضه العامة
- **chat/** — قائمة غرف + غرفة دردشة حية فعلياً عبر SSE (`useRoomSSE`)
- **dashboard/** — توجيه حسب الدور + لوحات تحكم منفصلة (أدمن/مندوب/عميل) تجلب إحصائيات فعلية عبر API
- **documents/** — رفع مستند وربطه بـ RFQ، وصفحة تفاصيل تعرض العناصر المستخرجة وتشغّل المعالجة
- **orders/** — قائمة الطلبات وتتبع الشحنة مع تحديث الحالة
- **pricing/** — حاسبة تكلفة كاملة، وCRUD لقواعد التسعير
- **profile/** — عرض/تعديل الملف الشخصي
- **quotes/** — قائمة عروض الأسعار وتفاصيلها (عرض/قبول/رفض/إنهاء)
- **rfq/** — إنشاء/قائمة/تفاصيل RFQ، بناء عرض سعر من RFQ + حاسبة تسعير، وصندوق وارد مطابقات للمورد
- **settings/** — ⚠️ **صفحة شبه فارغة (stub)** — لا يوجد فيها ربط بيانات فعلي حالياً

### ج. آليات جوهرية تم التأكد من عملها فعلياً في الكود

1. **محرك المطابقة الحصرية ثم البوابة العامة** (`intake/matcher.py`): يستخرج فئة المنتج من الكيانات المستخرجة بالـ AI، يطابقها مع الموردين حسب الفئة (تطابق مباشر بالكتالوج أو تداخل في ملف تعريف المورد)، يمنحهم نافذة حصرية مدتها **3 ساعات**، ثم يفتح الطلب للبوابة العامة (`is_public`) إن لم يُستجَب.
2. **الإشعارات الفورية عبر SSE** (وليست مجرد أيقونة واجهة): الباك إند يبث أحداث حقيقية (`new_rfq`, `quote_ready`) والفرونت إند (`useNotifications.ts`) يفتح اتصال `EventSource` حقيقي ويستهلكها فعلياً.
3. **مراجعة بشرية قبل نشر المنتجات (Human-in-the-loop)**: مُنفَّذة فعلياً — عمود `review_status` (pending/approved/rejected) في جدول المنتجات، والبحث العام في السوق يعرض فقط المنتجات المعتمدة (`review_status == approved`). المندوب يراجع من صفحة مخصصة قبل الظهور للعملاء.
4. **Token blacklist عند تسجيل الخروج**: مُنفَّذ فعلياً وبآلية مزدوجة — الـ refresh token يُبطَل عبر Redis blacklist بالـ jti، والـ access token يُبطَل عبر مقارنة توقيت إصداره بتوقيت "آخر تسجيل خروج" محفوظ في Redis (session invalidation).
5. **محرك التسعير عام بالكامل** — لا يوجد أي كود مرتبط بقطاع معين؛ يعمل بنفس الطريقة لأي منتج (سعر بالرنمينبي → تحويل عملة → شحن حسب الحجم → جمارك % → عمولة → ضريبة).
6. **لا توجد أكواد ناقصة/placeholder**: فحص شامل للباك إند عن TODO/FIXME/NotImplementedError/mock لم يُظهر أي نتائج — الموديولات الخلفية خالية من علامات "غير مكتمل".

## 4. ما لم يتم إنجازه فعلياً / نواقص حقيقية (مؤكدة من الكود الحالي)

1. **لا يوجد ربط مباشر (Foreign Key) بين الـ RFQ ومنتج الكتالوج.** جدول `rfqs` لا يحتوي على `catalog_product_id`، والعلاقة تمر بشكل غير مباشر عبر جدول `Product` (عناصر مستخرجة من طلب العميل) وليس عبر `CatalogProduct` (منتج المورد في الكتالوج) مباشرة. هذا ليس خطأً بل فجوة تصميم فعلية — يصعب على المندوب لاحقاً معرفة العنصر المحدد من الكتالوج الذي طلبه العميل بدقة 100%.
2. **صفحة الإعدادات (Settings) شبه فارغة** — مكوّن بسيط بدون أي ربط بيانات حقيقي حالياً.
3. **آلية الانتقال التلقائي من "حصري" إلى "عام" بعد انتهاء المهلة (3 ساعات)** — الاستعلام والموديل والفهرس (`ix_rfq_is_public_exclusive_deadline`) موجودون فعلاً، لكن لم يتم التأكد بشكل قاطع من وجود مهمة Celery Beat مجدولة تُنفّذ هذا الانتقال تلقائياً دون تدخّل — يحتاج تأكيداً إضافياً منفصلاً إن أردت.
4. بعض بنود الأمان منخفضة الأولوية (مثل TLS داخلي لتخزين MinIO، واستبدال `unsafe-inline` في CSP بـ nonces) لم يتم التحقق من إنجازها في هذه الجولة — تحتاج فحصاً مخصصاً إذا كانت أولوية.

## 5. الخلاصة

المنصة الآن **منتج عام متعدد الفئات** (ليست مقيدة بقطاع الإضاءة الصناعية كما كانت الوثائق القديمة تصفها)، والرحلة الكاملة تعمل فعلياً من الكود: رفع كتالوج → استخراج (OCR + AI) → **مراجعة بشرية قبل النشر** → ظهور في السوق العام → طلب عرض سعر → **مطابقة تلقائية حصرية ثم عامة** → إضافة شحن من المندوب → توليد PDF → **إشعار فوري حقيقي عبر SSE** → قبول العميل → تتبع الشحنة. الفجوة الحقيقية المتبقية الأبرز هي غياب الربط المباشر بين الـ RFQ ومعرّف منتج الكتالوج، بالإضافة لصفحة الإعدادات غير المكتملة.
